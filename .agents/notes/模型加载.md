# 模型加载

> prime-rl 中模型从 HuggingFace → GPU 的完整加载链路 + 推理服务启动。

## TL;DR

- prime-rl 用 **vLLM** 做推理后端
- 模型通过 `snapshot_download()` 从 HuggingFace Hub 下载，缓存于 `$HF_HOME`
- 权重广播有两种策略: **Filesystem**（默认）和 **NCCL**（量化场景）
- **无原生 S3 支持** — 需手动预下载到 cache 目录

---

## 1. 配置

### 模型名

```toml
[model]
name = "Qwen/Qwen3-0.6B"          # HuggingFace Hub ID
# 或
name = "/path/to/local/model"    # 本地路径（优先检查）
```

> 代码中通过 `Path(model_name).exists()` 判断是本地路径还是 Hub ID。

### 权重广播策略

```toml
[weight_broadcast]
type = "filesystem"  # 默认：从共享 FS 读
# 或
type = "nccl"       # 通过 NCCL 从 trainer broadcast（适合量化）
```

### 关键配置文件

| 配置 | 路径 |
|------|------|
| 简单单节点 | `configs/simple_inference.toml` |
| GLM-5 disaggregated | `configs/glm5_disagg_inference/inference.toml` |
| BlenderGym 训练 | `configs/multimodal/rl_blendergym_kaola.toml` |

## 2. 下载 & 缓存流程

```
User Config: model.name
    ↓
Path(model_name).exists()?
  YES → 直接用本地路径
  NO  → snapshot_download(repo_id=model_name)
         → 缓存到 $HF_HOME/hub/models--<org>--<model>/snapshots/<hash>/
    ↓
vLLM AutoModel.from_pretrained()
    ↓
GPU Memory (可用于推理)
```

### 关键代码

```python
# src/prime_rl/trainer/model.py
def pre_download_model(model_name: str) -> None:
    if Path(model_name).exists():
        return  # 跳过下载
    path = snapshot_download(repo_id=model_name, repo_type="model")
```

### 缓存层级

1. `HF_HOME` 环境变量（如 `/local-ssd/hf_cache`）
2. `~/.cache/huggingface/hub/`（默认）
3. 直接传给 `snapshot_download()` 的自定义路径

## 3. 权重广播

### Filesystem 策略（默认）

```
Shared Filesystem (/checkpoint/step_N/)
    ↓ (每个 vLLM worker 独立读)
GPU 0, GPU 1, ... (in-place copy)
```

```python
# src/prime_rl/inference/vllm/worker/filesystem.py
class FileSystemWeightUpdateWorker(Worker):
    def update_weights_from_path(self, weight_path: str):
        model_loader = get_model_loader(self.load_config)
        local_source = DefaultModelLoader.Source(weight_path, ...)
        weights_iterator = model_loader._get_weights_iterator(local_source)
        model.load_weights(weights_iterator)  # In-place → GPU
```

### NCCL 策略（量化场景）

```
Trainer (Rank 0) ──NCCL broadcast──→ Inference Ranks 1, 2, ...
                                      ↓
                                   GPU memory (in-place)
```

```python
# src/prime_rl/inference/vllm/worker/nccl.py
class NCCLWeightUpdateWorker(Worker):
    def init_broadcaster(self, host, port, rank_offset, ...):
        self.nccl_broadcast_receiver = NCCLWeightBroadcastReceiver(...)

    def update_weights_from_path(self, weight_dir: str):
        state_iter = self.nccl_broadcast_receiver.receive_state_dict()
        load_weights_checkpoint(model, state_iter)  # 或 load_weights_kernel (量化)
```

## 4. 推理服务启动

### 入口

```python
# src/prime_rl/entrypoints/inference.py
def main():
    config = cli(InferenceConfig)
    setup_vllm_env(config)
    from prime_rl.inference.vllm.server import server
    server(config)
```

### 服务初始化流程

```python
# src/prime_rl/inference/vllm/server.py
def server(config: InferenceConfig, ...):
    namespace = config.to_vllm()  # prime-rl config → vLLM args
    args = parser.parse_args(args=[], namespace=namespace)
    args.worker_extension_cls = WORKER_EXTENSION_CLS[config.weight_broadcast.type]
    # "filesystem" → FileSystemWeightUpdateWorker
    # "nccl" → NCCLWeightUpdateWorker
    uvloop.run(run_server(args))
```

### 运行命令

```bash
# 简单启动
uv run inference --model.name "Qwen/Qwen3-0.6B" --server.port 8000

# 从配置文件
uv run inference @ configs/glm5_disagg_inference/inference.toml

# 带 SLURM
uv run inference @ configs/... --slurm.partition gpu_cluster
```

## 5. Orchestrator 协调

```python
# src/prime_rl/orchestrator/orchestrator.py
async def orchestrate(config):
    inference_pool = await setup_inference_pool(client_config, model_name)
    await inference_pool.wait_for_ready(model_name)  # 等待模型加载完成
    # ...
    # 训练中更新权重:
    await inference_pool.update_weights(weight_dir, step=step)
```

两种 pool 模式:
- **Static**: 固定 URL 列表（`http://localhost:8000/v1`）
- **Elastic**: DNS 服务发现（动态扩缩）

## 9. 训练中权重更新流

```
1. Trainer 保存 checkpoint → /checkpoint/step_N/
2. Orchestrator → POST /update_weights {weight_dir: "/checkpoint/step_N"}
3. Inference server:
   - Filesystem 模式: 从共享 FS 读取 weights
   - NCCL 模式: 从 trainer rank 0 接收 broadcast
4. Model weights in-place 更新，继续推理
```

## 11. 常见问题

| 问题 | 解决 |
|------|------|
| Model not found | 检查 Hub ID / 网络 / HF_TOKEN |
| CUDA out of memory | 降低 `gpu_memory_utilization`，增加 `tp` |
| 启动慢 "Waiting for inference pool" | 首次需下载模型（大模型可能需数小时） |
| S3 模型加载 | 预下载: 设 `HF_HOME=/local-ssd/hf-cache`，预填充 cache |

---

## 12. 性能建议

- 用 local SSD 做 cache: `export HF_HOME=/local-ssd/hf-cache`
- 大模型用 TP: `tp = 4` 分布到多 GPU
- 预下载模型，避免训练时等待
- Filesystem broadcast 适合共享存储；NCCL 适合量化权重

### A2. 下载 & 缓存管线

```
User Config: model.name = "Qwen/Qwen3-0.6B"
                  ↓
                  ├─ Path(model_name).exists()?
                  │  YES → 直接用本地路径
                  │  NO  ↓
                  │
                  └─ snapshot_download(repo_id=model_name)
                     │
                     └─ $HF_HOME/hub/models--<org>--<model>/snapshots/<hash>/
                        │
                        ├─ 已缓存? → 返回 cache path
                        │
                        └─ 未缓存 → 从 HF Hub 下载
                           (safetensors + config.json)
                           ↓
                           缓存到本地 → 返回 path
                                       ↓
                  vLLM AutoModel.from_pretrained()
                           ↓
                  GPU Memory → Ready
```

### A3. Tensor Parallel 加载 (多 GPU)

```
Config: tp=4, model="Qwen/Qwen3-32B"

Local Cache (8 个 safetensors shards):
├── model-00001-of-00008.safetensors
├── model-00002-of-00008.safetensors
├── ...
└── config.json
     │
     └─ vLLM Model Loading (TP=4)
        ├─ Rank 0 读 shards 1-2 → GPU 0
        ├─ Rank 1 读 shards 3-4 → GPU 1
        ├─ Rank 2 读 shards 5-6 → GPU 2
        └─ Rank 3 读 shards 7-8 → GPU 3
                                    ↓
                    Model ready (TP=4)
```

### A4. Filesystem 权重广播

```
[初始加载]
Shared Filesystem (/checkpoint/step_N/)
    ↓ (每个 vLLM worker 独立读)
    ├─ Inference Node B: GPU 0-3 ← read weights
    └─ Inference Node C: GPU 0-3 ← read weights

[训练中更新]
Training Node A
  └─ Save checkpoint → /checkpoint/step_2000/
                            ↓
     Orchestrator: POST /update_weights {weight_dir: "..."}
                            ↓
     Inference Nodes: FileSystemWeightUpdateWorker
       └─ Read from FS → in-place copy → GPU memory
                            ↓
     Ready with new weights

✅ 简单，无需节点间数据拷贝
❌ 需要共享文件系统 (NFS/GCS/S3)
```

### A5. NCCL 权重广播

```
[Setup]
Trainer Master (Rank 0)
  └─ Start NCCL server: host=trainer, port=29500, rank=0
                            ↓
Orchestrator → init_broadcaster RPC → Inference Ranks
  ├─ Rank 1: connect(host=trainer, port=29500, world_size=3)
  └─ Rank 2: connect(host=trainer, port=29500, world_size=3)

[Weight Transfer]
Trainer (Rank 0)
  └─ Serialize state_dict → NCCL broadcast
                            ↓
     ┌────────────┬─────────────────┐
     │            │                 │
  Rank 0      Rank 1            Rank 2
  (Train)     (Infer)           (Infer)
                ↓                 ↓
          GPU load weights   GPU load weights
          (in-place)         (in-place)

✅ 无需共享文件系统，支持量化权重
❌ 需要 TCP 连接，setup 更复杂
```

```
              REQUEST ROUTER (vllm-router, Port 8000)
                    │
        ┌───────────┴──────────────┐
        ↓                          ↓
 PREFILL NODES 0-3          DECODE NODES 4-5
```

### A7. Orchestrator ↔ Inference 协调

```
[启动阶段]
Orchestrator                         Inference Servers
├─ load config                       │ (idle)
├─ setup_inference_pool()            │
├─ wait_for_ready(model_name) ─────► │ GET /v1/models
│                                    │ snapshot_download() if needed
│                              ◄──── │ Health OK
└─ Create Scheduler                  │ Ready

[训练循环]
Orchestrator                         Inference Servers
├─ Request rollouts ────────────────► POST /v1/chat/completions
│                              ◄──── Return completions
├─ Save checkpoint                   │
├─ POST /update_weights ────────────► Load from FS/NCCL
│                              ◄──── OK
└─ Continue next step                │ Ready (new weights)
```

### A8. 多节点分布

```
100B 参数模型, 4 nodes, TP=8

Node 0 (TP rank 0-7)     Node 1 (rank 8-15)
┌──────────────┐          ┌──────────────┐
│ GPU 0..7     │          │ GPU 0..7     │
│ Shard 1 (25%)│          │ Shard 2 (25%)│
└──────────────┘          └──────────────┘

Node 2 (rank 16-23)      Node 3 (rank 24-31)
┌──────────────┐          ┌──────────────┐
│ GPU 0..7     │          │ GPU 0..7     │
│ Shard 3 (25%)│          │ Shard 4 (25%)│
└──────────────┘          └──────────────┘

加载: 每个 rank 从 shared cache 读对应 shard
推理: All-Reduce (NCCL over TCP)
```

### A9. Cache Hit vs Miss

```
[Cache Miss — 首次运行]
model.name = "zai-org/GLM-5-FP8"
  → Path exists? No
  → snapshot_download()
  → Check cache: NOT FOUND ✗
  → Download from HF Hub (81 files, ~1TB, 数小时)
  → Save to $HF_HOME/hub/
  → vLLM load → GPU → Ready (漫长等待后)

═══════════════════════════════════════════

[Cache Hit — 后续运行]
model.name = "zai-org/GLM-5-FP8"
  → Path exists? No
  → snapshot_download()
  → Check cache: FOUND ✓
  → Return cache path (~5s)
  → vLLM load → GPU → Ready (快速启动)

💡 关键: export HF_HOME=/local-ssd/hf-cache (NVMe 加速 IO)
```
